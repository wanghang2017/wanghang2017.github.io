    # babel篇

    # 简史

    08年 v8诞生 js运行速度迅速提升

    09年 node诞生 前端工程化基础

    10年 npm出现 前端包管理基础

    14年 6to5发布 使用 acorn 做AST引擎

    15年 6to5+Esnext => babel 解析引擎Babylon 后来Babylon移入到@babel/parser

    # 组成

    @babel/core AST转换的核心

    @babel/cli 打包工具

    @babel/preset-env* 插件的集合 减少配置

    @babel/plugin* babel插件机制 Babel基础功能不满足时 手动添加些

    @babel/runtime* 类似polyfill 但是 按需导入代码少

    @babel/polyfill 把浏览器不支持的api 兼容性代码全部导入项目 不管用到用不到 代码量大

    .babelrc/babel.config.js 配置文件

    # @babel/core

    babel编译器。被拆分三个模块：@babel/parser，@babel/traverse，@babel/generator。

    @babel/parser: 接受源码，进行词法分析、语法分析，生成AST。

    @babel/traverse：接受一个AST，并对其遍历，根据preset、plugin进行逻辑处理，进行替换、删除、添加节点。

    @babel/generator：接受最终生成的AST，并将其转换为代码字符串，同时此过程也可以创建source map。

    babel转码流程：input string -> @babel/parser parser -> AST -> transformer[s] -> AST -> @babel/generator -> output string。

    # .babelrc/babel.config.js

    核心：presets 和 plugins

    {
    "presets": [
        [
            "@babel/preset-env",
            {
                "debug": true,
                "targets": ["last 2 versions"]
            }
        ]

    "plugins": [
        ["@babel/plugin-transform-runtime"]
    ],
}

    presets是plugins的集合

    最常用的是@babel/preset-env

    vue用的@vue/app = @babel/preset-env + 自定义的plugins

    参数：

    debug：babel的日志信息，可以看到哪些语法做了转换 默认false

    useBuiltIns: 取值 false entry usage 默认false啥也不干，entry 引入所有polyfill，usage用到的polyfill引入。

    targets： 用来指定转换时需要哪些浏览器支持

    # presets和plugins的加载顺序

    plugins优于presets加载

    plugins顺序加载

    presets倒序加载

    # polyfill和runtime的区别

    babel只是转换syntax层语法 所以需要 @babel/polyfill 来处理API

    又因为polyfill体积太大 所以通过preset的useBuiltIns来实现按需加载

    再接着为了npm组件开发需要， 出现了@babel/runtime来做隔离
