    # 项目文档

    # 1. 初始化项目

    npx create-react-app jira --template typescript

    # 2.文件说明

    src>react-app-env.d.ts 为我们引入一些预先定义好的typescript的类名？

    src>reportWebVitals.ts 做埋点上报

    src>setupTests.ts 配置单元测试

    public>manifest.json 用于做pwa 配置pwa加载情况

    public>robots.txt 用于配置 搜索引擎爬虫怎么对待我们的页面

    # 3.项目工具

    3.1 prettier

    # 4. mock方案

    1. 代码入侵（直接写死mock数据 或者请求本地的JSON文件）
    2. 请求拦截 （代表Mock.js） 优： 代码分离 可生成随机数 缺：无法真实模拟增删改查 只支持ajax 不支持fetch
    3. 接口管理工具 （rap swagger moco yapi） 优： 配置功能强大 接口管理与mock一体 后端修改接口 Mcok也跟着更改 可靠 缺：配置复杂 依赖后端 大团队使用较多 人少慎重考虑
    4. 本地 node服务器 json-server 优： 配置简单 可自定义 增删改查真实模拟 缺： 无法随着后端api的修改而自动修改

    # 5. REST API

    URI代表资源/对象 METHOD代表行为

    GET /tickets //列表

    GET /tickets/12 // 详情

    POST /tickets // 增加

    PUT /tickets/12 // 替换

    PATCH /tickets/12 // 修改

    DELETE /tickets/12 //删除

    # 6. json-server

    yarn add -D json-server

    再package.json中配置 json-server --watch --port 53000 __ json_server_mock__/db.json

    # 7.hooks

    # 8.ts

    TypeScript 的8种类型： number string boolean 函数 array any void object

    1. number 数字类型 包含小数 其它进制的数字
    2. string 字符串
    3. array 在ts中 array一般指 所有元素类型相通的值的集合
    4. 在TS中类型不同的数组 叫做 tuple
    5. boolean 布尔值
    6. 函数 1.js函数上直接声明参数和返回值 2.直接声明你想要的函数类型
    7. any 表示这个值可以是任何值 被定义为any就意味着不做任何类型检查
    8. void 绝大部分情况下 只会用在这一个地方 ： 表示函数不反回任何值或者返回undefined
    9. object 除了number string boolean bigint symbol null undefined 其它都是object
    10. tuple 数量固定 类型可以各异
    11. enum
    12. null 和 undefined 是值 也是类型
    13. unknown 表示这个值可以是任何值 是严格版的any 不能复制给任何类型 也不能读取任何属性
    14. never const func = ()=>{ throw new Error() }
    15. interface 不是一种类型 应该被翻译成接口 或者说使用上面介绍的类型 创建一个我们自己的类型
    16. 啥时候声明类型？
    17. 理论上来说 我们声明任何变量的时候都需要声明类型（包括普通变量 函数 组件 hook等等），声明函数 组件 hook等需要声明参数 和 返回值类型 但是很多情况下 ts可以帮我们自动推断 我们不用声明
    18. .d.ts js文件+.d.ts文件 === ts文件 .d.ts文件可以让js文件继续维持自己js文件的身份，而拥有TS的类型保护 一般我们写业务代码不会用到 但是点击类型跳转一般会跳到d.ts文件

    # ts Utility Types

    1. 联合类型

    let num : string | number

    1. 类型别名 类型别名在很多情况下可以和 interface 互换

    type FavoriteNumber = string|number

    区别： 定义联合类型 交叉类型 使用type interfaca没法实现Utility Type

    1. utility type 的用法 : 用泛型给它传入一个其它类型 然后 utility Type 会对该类型进行操作

    Parameter => 提取xx属性

    Partial = {} => 让必须的属性 变成可有可无

    Omit<Person,'name'|‘age’> = {} => 删除指定类型的指定属性
